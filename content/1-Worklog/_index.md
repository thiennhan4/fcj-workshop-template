**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with AWS and basic AWS services](1.1-week1/)

**Week 2:** [Studied and completed the lab exercises for each training module.](1.2-week2/)

**Week 3:** [Learned additional knowledge about AWS networking architecture and Amazon VPC.](1.3-week3/)

**Week 4:** [Practiced deploying Amazon EC2 instances and explored Infrastructure as Code (IaC) using AWS CloudFormation.](1.4-week4/)

**Week 5:** [Researched Amazon VPC Peering and practiced configuring VPC Peering.](1.5-week5/)

**Week 6:** [Learned about Amazon EBS, Amazon EFS, and other storage services, and practiced using AWS Backup for data protection.](1.6-week6/)

**Week 7:** [Researched Amazon S3 and other AWS services.](1.7-week7/)

**Week 8:** [GStudied and completed hands-on lab exercises.](1.8-week8/)

**Week 9:** [Analyzed project requirements and created the initial project framework for the capstone project.](1.9-week9/)

**Week 10:** [Developed the overall project concept, designed preliminary features, defined the system architecture, and created an initial cloud architecture design for the project.](1.10-week10/)

**Week 11:** [Initialized the backend project, connected the database, and implemented entities, DTOs, migrations, and other foundational components.](1.11-week11/)

**Week 12:** [Researched deployment strategies and system testing methods.](1.12-week12/)
